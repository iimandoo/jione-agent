## Steps

1. **Theme 토큰 정의**
   - `src/styles/theme.ts` 생성 및 위 토큰 작성
   - TypeScript로 타입 안정성 확보

2. **ThemeProvider 생성**
   - `src/styles/provider.tsx` 생성
   - styled-components의 `ThemeProvider` 설정
   - Root Layout에 감싸기

3. **globals.css 정리**
   - CSS 리셋 및 기본 스타일만 정의

4. **컴포넌트 스타일 작성**
   - 각 섹션/컴포넌트를 styled-components로 작성
   - 토스 디자인 토큰 참조

5. **빌드 확인**
   ```bash
   cd jione-portfolio && npm run build
   ```
   빌드 에러가 있으면 수정 후 재시도

#### Step 2: `src/styles/provider.tsx` 생성

```typescript
'use client';

import React, { useState } from 'react';
import { useServerInsertedHTML } from 'next/navigation';
import { ServerStyleSheet, StyleSheetManager, ThemeProvider } from 'styled-components';
import { theme } from './theme';

// SSR 스타일 주입 (FOUC 방지) — Next.js App Router + styled-components v6 필수
export function StyleProvider({ children }: { children: React.ReactNode }) {
  const [styledComponentsStyleSheet] = useState(() => new ServerStyleSheet());

  useServerInsertedHTML(() => {
    const styles = styledComponentsStyleSheet.getStyleElement();
    styledComponentsStyleSheet.instance.clearTag();
    return <>{styles}</>;
  });

  if (typeof window !== 'undefined') {
    return <ThemeProvider theme={theme}>{children}</ThemeProvider>;
  }

  return (
    <StyleSheetManager sheet={styledComponentsStyleSheet.instance}>
      <ThemeProvider theme={theme}>{children}</ThemeProvider>
    </StyleSheetManager>
  );
}
```

## Rules

- **테마 일관성**: 모든 컴포넌트는 `styled-components`와 `theme` 객체 사용
- **기존 콘텐츠**: 텍스트, 이미지 경로, data.ts는 절대 변경하지 않음
- **빌드 성공**: 반드시 `npm run build` 실행 후 에러 없음 확인
- **TypeScript**: 타입 안정성을 위해 `styled.ts` 파일 생성 시 타입 정의 필수
- **반응형**: mobile-first 방식으로 media queries 작성
