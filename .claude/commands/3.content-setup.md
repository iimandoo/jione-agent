🟢 **Phase 1 — 콘텐츠 설정** (필수)

포트폴리오에 표시될 모든 콘텐츠를 `data/resume.ts` 단일 파일로 관리합니다.

⚠️ **사전 조건**: `/project-init` 명령어가 먼저 실행되어야 합니다.

## 목표

- `data/resume.ts` — 경력, 프로젝트, 기술, SEO, 임팩트를 담은 **단일 소스**
- 모든 섹션 컴포넌트가 이 파일 하나만 참조

## Steps

### 1. `data/resume.ts` 존재 여부 확인 (필수)

아래 순서대로 Read 툴로 파일을 확인하세요.

**① 먼저 `data/resume.ts` (portfolio-agent 루트) 읽기 시도**

- **파일이 있으면 → 해당 내용을 `jione-portfolio/src/data/resume.ts`에 복사한 뒤 Step 2로 진행**

**② 없으면 `jione-portfolio/src/data/resume.ts` 읽기 시도**

- **파일이 있으면 → Step 2로 진행**

**③ 두 파일 모두 없으면 → 작업을 즉시 중단**하고 아래 메시지를 출력:

```
❌ resume.ts 파일을 찾을 수 없습니다.

아래  위치에 파일을 생성한 뒤 다시 실행해주세요:
  - data/resume.ts
```

---

### 2. `data/resume.ts` 구조 검증 및 보완

읽은 파일의 최상위 섹션을 확인합니다. 아래 섹션 중 누락된 항목이 있으면 파일 끝(`) as const` 직전)에 추가합니다.

| 섹션             | 키        | 필수 여부 |
| ---------------- | --------- | --------- |
| SEO & 메타데이터 | `meta`    | 필수      |
| 기본 프로필      | `profile` | 필수      |
| 경력 타임라인    | `career`  | 필수      |
| 기술 스택        | `skills`  | 필수      |
| 프로젝트         | `Project` | 필수      |

누락된 섹션은 아래 **전체 구조 레퍼런스**의 샘플 데이터로 채웁니다.

### 4. 컴포넌트에서 사용 예시

모든 섹션 컴포넌트는 `resume`를 직접 import합니다. 별도 로더 불필요.

**`src/components/sections/about.tsx`** 예시:

```typescript
'use client';

import styled from 'styled-components';
import { resume } from '@/data/resume';

const AboutSection = styled.section`
  padding: 5rem 1.5rem;
`;

const Container = styled.div`
  max-width: 64rem;
  margin: 0 auto;
`;

const SectionTitle = styled.h2`
  font-size: 1.875rem;
  font-weight: 700;
  margin-bottom: 2rem;
  color: ${(props) => props.theme.colors.foreground};
`;

const ExperienceCard = styled.div`
  background: #fff;
  padding: 1.5rem;
  border-radius: ${(props) => props.theme.radius.lg};
  box-shadow: ${(props) => props.theme.shadows.sm};
  margin-bottom: 1.5rem;
`;

export function About() {
  const { career } = resume;

  return (
    <AboutSection>
      <Container>
        <SectionTitle>경력</SectionTitle>
        {career.experiences.map((exp) => (
          <ExperienceCard key={exp.id}>
            <h3>{exp.position}</h3>
            <p>{exp.company} · {exp.period.start} ~ {exp.period.end}</p>
            <p>{exp.description}</p>
          </ExperienceCard>
        ))}
      </Container>
    </AboutSection>
  );
}
```

**`Project` 섹션 사용 예시** (별도 섹션 컴포넌트로 분리하거나 about에 포함):

```typescript
'use client';

import styled from 'styled-components';
import { resume } from '@/data/resume';

const ProjectSection = styled.section`
  padding: 5rem 1.5rem;
  background-color: ${(props) => props.theme.colors.muted};
`;

const Container = styled.div`
  max-width: 64rem;
  margin: 0 auto;
`;

const Intro = styled.p`
  font-size: 1.0625rem;
  line-height: 1.7;
  color: ${(props) => props.theme.colors.secondaryForeground};
  margin-bottom: 3rem;
  max-width: 48rem;
`;

const CaseCard = styled.div`
  background: ${(props) => props.theme.colors.card};
  padding: 1.75rem;
  border-radius: ${(props) => props.theme.radius.lg};
  box-shadow: ${(props) => props.theme.shadows.sm};
  margin-bottom: 1.5rem;
`;

export function ProjectCasesSection() {
  const { Project } = resume;

  return (
    <ProjectSection id="Project">
      <Container>
        <Intro>{Project.intro}</Intro>
        {Project.cases.map((c) => (
          <CaseCard key={c.id}>
            <h3>{c.title}</h3>
            <p>{c.company}</p>
            <p>{c.Project}</p>
            <p>{c.execution}</p>
            {'tools' in c && c.tools && <p>Tools: {c.tools}</p>}
          </CaseCard>
        ))}
      </Container>
    </ProjectSection>
  );
}
```

## Rules

- **단일 소스**: 콘텐츠 수정은 오직 `data/resume.ts` 하나만 변경
- **JSON 금지**: 별도 JSON 파일 생성하지 않을 것
- **파일 없으면 중단**: `data/resume.ts`가 없으면 작업을 시작하지 않음
- **덮어쓰기 금지**: 기존 파일의 내용을 덮어쓰지 않고 누락 섹션만 추가
- **타입 안정성**: `as const` + 추출 타입으로 컴파일 타임 검증
- **이미지 경로**: 모두 `public/images/` 상대 경로 사용
- **기간 형식**: "YYYY-MM" 형식 통일
